NOTES & KNOWN ISSUES
---------------------
1. TEXT-ONLY PDFs (C8455 AND MA-INJ-6)
   Neither the C8455 nor the MA-INJ-6 datasheet has structured
   tables — all data is in flowing text. Specs were extracted
   using regex pattern matching on section headings and key-value
   text pairs. This works well for clearly formatted lines but
   may miss values if the PDF text has unusual spacing (especially
   for the C8455 where some spec lines ran together in extraction).

2. C8455 DIMENSIONS
   The "Dimensions (h x w x d)" label and value appeared on
   separate lines in the raw text extraction, which is a common
   pdfplumber artifact for multi-line labels. The value was
   successfully captured as "1.73 in x 17.50 in x 12 in".

3. C8455 SFP COMPATIBILITY TABLES (PAGES 3-4)
   The SFP compatibility tables have 3 columns (Module, 25G ports,
   10G ports, 1G ports). Since pdfplumber could not parse these
   as proper tables, SFP module SKUs were extracted as a flat
   list from the raw text section. Port-level compatibility
   (which module works on which port) is NOT captured — this
   would require a more complex custom parser.

4. CAMPUS GATEWAY SUPPORTED SFPs (PAGES 6-7)
   The full SFP compatibility list spans two pages. The combined
   list was extracted from the raw table values across both pages
   and stored as a flat list in the JSON. This covers all
   SFP, SFP+, and SFP28 modules listed in the document.

5. IMAGES FIELD IS EMPTY
   All three PDFs have no downloadable image URLs.
   "images": [] for all products.

6. URLS ARE CONSTRUCTED
   URLs are best-guess patterns and have not been verified live.
